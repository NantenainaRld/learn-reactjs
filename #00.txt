# 00
- ReactJS is a library created by Meta (formerly Facebook).
- Prerequisites:
    - JavaScript proficiency (advanced).
    - Node.js installed on your system.
    - Mastery of JavaScript ES6+ syntax (Arrow functions, Destructuring, Modules, etc.).

## Initialization with 2 methods:
    - Using CRA (Create React App):
    ```bash
    npx create-react-app my-react-app
    ```

    - Using Vite (Recommended):
        ```bash
        npm create vite@latest
        # or
        npx create-vite@latest
        ```
        - Follow the prompts (Select React, then JavaScript/JSX).

## Hello World:
    - Edit the `src/App.jsx` file and save.
    - The changes are applied automatically via **HMR** (Hot Module Replacement) without a full page reload.      

## Folder structure:
    - **package.json**:
        - Contains project metadata, dependencies, and scripts (`dev`, `build`, `preview`).
    - **node_modules**:
        - Contains all installed dependencies (never modify this folder manually).
    - **index.html**:
        - The entry point for the browser. In Vite, it's at the root and contains the `<div id="root"></div>` where React injects the UI.
    - **public/**:
        - Contains static assets (images, icons) that don't need processing by Vite.
    - **src/main.jsx**:
        - The JavaScript entry point. Uses `createRoot` from `react-dom/client` to render the App.
    - **src/App.jsx**:
        - The main component of your application.
    - **vite.config.js**:
        - Configuration file for Vite (plugins, server ports, etc.).

# 02
## Components:

- **Reusability**: 
    - Components are independent and reusable bits of code. While React components are specific to React, the concept of "Web Components" exists for cross-framework use.
- **Functional vs Class Components**:
    - **Functional Components (Modern/Stateless-like)**: 
        - These are simple JavaScript functions that return JSX. In 2026, they are the standard.
        - Even if they hold state (via Hooks), they are still written as functions.
    - **Class Components (Legacy/Stateful)**: 
        - Use ES6 classes that extend `React.Component`. 
        - They are now considered legacy and rarely used in new React 19+ projects.

- **Example (Functional Component)**:
    ```jsx
    function Welcome() {
        return <h1>Hello, React 19!</h1>;
    }
    ```

## Create our first functional component:

- Create the folder `src/components`.
- Create the file `src/components/Greet.jsx`.
- Define the component:
    ```jsx
    function Greet() {
        return <h1>Hello from Greet Component!</h1>;
    }
    export default Greet;
    ```
- Import and use the component in `src/App.jsx`:
    ```jsx
    import Greet from './components/Greet';

    function App() {
        return (
            <>
                <Greet />
                {/* Or with children if handled by props */}
                <Greet>Text here</Greet>
            </>
        );
    }
    ```

## Create our first class component (Legacy/Not recommended):

- Create the file `src/components/Welcome.jsx`.
    ```jsx
    import { Component } from 'react';

    class Welcome extends Component {
        render() {
            return <h1>Welcome from a Class Component!</h1>;
        }
    }
    export default Welcome;
    ```

## Rules and Exports:

- A component must return a single parent element (use Fragments `<>` if needed).
- Default Export: 
    - Usage: `export default Greet;`
    - Import: `import Greet from './components/Greet';` (You can rename the variable during import).
- Named Export: 
    - Usage: `export const Greet = () => ...;`
    - Import: `import { Greet } from './components/Greet';` (Must use the exact name inside curly braces).

# 03
## Props:
- Props are attributes passed to components, similar to function arguments.
- Example:
    - Greet.jsx:
        ```jsx
        const Greet = ({ title }) => {
            return <h1> {title} </h1>;
        };
        export default Greet;
        ```
    - App.jsx:
        ```jsx
        <Greet title="Props example" />
        ```

## Children:
- The `children` prop allows you to pass elements or text between the opening and closing tags of a component.
- Example:
    - Greet.jsx:
        ```jsx
        const Greet = ({ children }) => {
            return <div> Content: {children} </div>;
        };
        export default Greet;
        ```
    - App.jsx:
        ```jsx
        <Greet> This is the children content </Greet>
        ```

## Important Notes:
- Props are read-only (immutable). A component should never modify its own props.
- You can destructure props directly in the function arguments for cleaner code.
- Fragments `<>` can be used if you need to return multiple elements alongside props/children.
